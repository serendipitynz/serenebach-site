# Serene Bach Site

Landing page and public website for [Serene Bach](https://github.com/serendipitynz/serenebach), a self-hosted weblog engine built with Go, SQLite, static rebuilds, MCP, and AI-assisted writing.

> Japanese: see [README.ja.md](README.ja.md)

## Stack

- [Astro](https://astro.build/) for static site generation
- SCSS for styling
- `pnpm` for package management

## Pages

| Path | Purpose |
|---|---|
| `/` | English landing page |
| `/ja/` | Japanese landing page |
| `/en/` | Compatibility redirect to `/` |
| `/templates/` | English template gallery |
| `/ja/templates/` | Japanese template gallery |
| `/news/` | News pages, generated by the Serene Bach engine (see [News](#news)) |

English is the primary language for the public site. Japanese remains a first-class page because Serene Bach has a Japanese SB3 / CGI hosting background.

The landing page surfaces the latest 3 news entries by reading `public/news/atom.xml` at build time via `src/data/news.ts`.

## Development

```bash
pnpm install
pnpm dev
```

Astro defaults to <http://localhost:4321/>. If the port is already in use, it will choose another port.

`predev` runs `scripts/build-templates.mjs` before the dev server starts, so the template gallery has fresh ZIPs and metadata.

## Build

```bash
pnpm build
```

`prebuild` runs `scripts/build-templates.mjs`. The static output is written to `dist/`.

## Templates

Template packages live under `templates/<slug>/` and must include:

- `<slug>.txt` with `Name:`, `Author:`, `Version:` fields (and optional `Address:`)
- `<slug>.jpg` as a light-mode thumbnail
- Optional `<slug>-dark.jpg` for the dark-mode thumbnail

`templates/order.json` is an array of slugs that should appear first in the gallery. Slugs not listed there fall back to alphabetical order. Missing pinned slugs produce a warning during the build.

`scripts/build-templates.mjs`:

- Reads each template directory and writes a ZIP to `public/templates/<slug>/<slug>_<version>.zip`
- Copies the JPG thumbnails to `public/templates/<slug>/`
- Generates English / Japanese `readme.md` files inside each source directory (gitignored)
- Writes the gallery metadata to `src/data/generated/templates.json`

Both `public/templates/` and the per-template `readme*.md` files are gitignored — they are always regenerated by the build script.

## News

The news section under `/news/` is produced by the Serene Bach engine itself, not by Astro:

- Markdown source lives in `content/` (one English + one Japanese file per release)
- `config/` contains the Serene Bach News templates (`top.txt`, `category_en.txt`, `category_ja.txt`) and the category-to-template mapping documented in [config/README.md](config/README.md)
- `scripts/update-news.sh` runs the `serenebach` CLI to import Markdown from `content/` and rebuild static HTML / `atom.xml` / `rss.xml` into `public/news/`
- Pass `publish` (`scripts/update-news.sh publish`) to also commit and push the regenerated `content/` and `public/news/` trees

The script expects a `serenebach` binary on `PATH` and a local sandbox under `_sandbox/data/` (gitignored).

## Screenshots

Screenshots are stored under `public/screenshots/` and have light / dark variants:

```text

admin-ai-settings-dark.png

```

`admin-tour.mp4` and `admin-tour-poster.jpg` are also stored here for the landing page tour video.

The site switches screenshot variants automatically with `prefers-color-scheme`. The same rule is used for the lightbox preview and the template gallery thumbnails.

## Open Graph

The shared social preview image is `public/sb_opengraph.png` at `1200 x 630`.

`BaseLayout.astro` emits Open Graph and Twitter Card metadata. The production URL is configured as `https://go.serenebach.net` in `astro.config.mjs`, so `og:image` is emitted as an absolute URL.

## Verification

Before finishing changes, run:

```bash
pnpm build
```

`astro build` performs the same type-aware checks Astro uses for type-checking `.astro` files. TypeScript itself is not a direct dev dependency of this project; if you want stricter standalone checks, install `@astrojs/check` and run `pnpm exec astro check`.
